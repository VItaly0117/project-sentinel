# Session Note

## What changed
- Added a small `sentinel_training.ingest` package with explicit source-specific loaders for Binance and Bybit.
- Added a shared normalization layer that validates rows and writes deterministic normalized CSV outputs in `data/normalized/`.
- Added sidecar metadata JSON with source, symbol, interval, row count, timestamp range, and input/output SHA-256 hashes.
- Added focused pytest coverage for Binance ZIP parsing, Bybit JSON parsing, duplicate timestamp rejection, malformed rows, and numeric coercion failures.
- Updated `README.md`, `docs/training-data-sources.md`, `ai/current-state.md`, and `ai/progress.md` for the new ingest workflow.

## What decisions were made
- Kept ingestion local-first: the CLI normalizes saved files and does not download from exchanges yet.
- Kept Binance and Bybit paths separate by default so venue comparison stays clean.
- Chose explicit parsers over a generalized market-data abstraction layer.

## Open risks
- There is still no downloader/paginator for multi-month exchange backfills.
- Binance CSV layout assumptions are based on the standard bulk kline shape; unusual custom exports may still fail validation.
- Bybit support is intentionally narrow around saved V5 kline-style payloads, not every possible export format.
- Training research still uses OHLC-based labels and ignores microstructure effects.

## Next recommended task
- Run the new ingest CLI on one real Binance archive and one saved Bybit response, then train two separate baseline datasets and compare artifacts without mixing venues.
