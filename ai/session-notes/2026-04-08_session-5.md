# Session Note

## What changed
- Tightened ingest timestamp parsing to require Unix milliseconds for the supported Binance and Bybit candle inputs.
- Fixed Binance ZIP provenance hashing to fingerprint extracted CSV content instead of the ZIP container.
- Fixed Binance parsing to drop repeated embedded header rows inside local CSV/ZIP files.
- Added deterministic float formatting for normalized CSV writes.
- Added Bybit alias-conflict detection and expanded ingest pytest coverage for the new safeguards.

## What decisions were made
- Kept the ingest MVP strict and explicit instead of supporting multiple timestamp units or broader source guessing.
- Preserved separate Binance and Bybit output paths and did not add downloading, pagination, or multi-source orchestration.
- Treated ambiguous Bybit column aliases as hard errors instead of silently selecting one.

## Open risks
- The ingest layer still expects a narrow set of real source shapes: Binance bulk-style klines and saved Bybit V5 kline-style payloads.
- There is still no automated downloader/paginator for historical backfills.
- Training research still uses OHLC-based labels and ignores microstructure effects.
- Partial-output recovery is still manual if an operator interrupts a write and reruns without `--overwrite`.
