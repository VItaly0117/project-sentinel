# Session Note

## What changed
- Added startup reconciliation between persisted runtime markers and exchange open exposure.
- Persisted `last_action_candle_time`, `last_action_side`, and `last_action_order_id` in SQLite runtime state.
- Added runtime-level duplicate action skipping for the same candle/action and SQLite-level dedupe for duplicate signal/trade inserts.
- Extended runtime tests for matched restart reconciliation, ambiguous restart failure, duplicate action skipping, and persisted action markers.
- Added README inspection commands for `runtime_state` and `startup_reconciliation*` events.

## What decisions were made
- Kept reconciliation explicit and conservative: continue only on clear order-id match or single-position side match; otherwise stop.
- Stored restart/idempotency markers inside the existing `runtime_state` table instead of adding new tables or ORM abstractions.
- Logged reconciliation mismatches/events into existing `runtime_events` and `error_events` tables for operator inspection.
- Preserved the current runtime module split and kept `sentineltest.py` unchanged as a thin entrypoint.

## Open risks
- Reconciliation still depends on Bybit response quality and limited exchange-side identifiers; complex manual intervention cases remain ambiguous by design.
- Local SQLite is still single-node persistence only.
- Runtime still depends on a local `monster_v4_2.json`.
- Training pipeline still has no automated tests.
